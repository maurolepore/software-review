2. Ejercitar el proceso para proponer software para revision

Proponer software para revision: Abrir issue

Abri un issue para proponer software en el repositorio de practica. Podes usar tu paquete o cualquiera de estos: eph, karel, agroclimatico.

Proponer software para revision: Communicar amablemente

Como editor/a, escribir un comendario constructivo y amable notificando que el paquete esta fuera del alcance de rOpenSci.

A modo de ejemplo podes usar karel, que fue revisado como exepcion por el programa de campeopne/as pero que normalmente esta fuera del alcancd de las categorias de rOpenSci.

Comunicacion: Estructura

La estructura del feedback constructivo y amable

  • Como revisor/a, escribi una introduccion minima pero constructiva y amable como encabezado para una revision imaginaria.

  • Como autor/a, escribi una introduccion minima pero constructiva y amable como encabezado para tus respuestas imaginarias a las sugerencias de un/a revisor/a.

Repaso

Escribir reflexiones que cuenten la historia de lo que vimos

  • Que? ej. Las actividades principlaes del proceso son …
  • Quien? ej. Los roles que participan en la pre-submision son …
  • Como? ej. El feedback debe ser …
  • Donde? ej. Todo el proceso ocurre en ….
  • Por que? e.g. La revision por pares ayuda a …
  • Cuando? ej. La evaluacion de alcance debe hacerse antes de …
